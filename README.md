# 题库练习

在线答题器，支持多题库、多模式、错题本管理、答题分析。

## 部署

将以下文件放到同一目录，推送到 GitHub Pages 即可：

- `index.html` — 页面结构
- `style.css` — 样式
- `app.js` — 逻辑
- `favicon.svg` — 图标
- `*.json` — 题库数据
- `*.ogg` — 音效文件（可选）

## 添加题库

1. 将 JSON 文件放到同目录
2. 在 `app.js` 顶部 `QUIZ_SOURCES` 数组添加一行：

```js
const QUIZ_SOURCES = [
  { file: 'Marxism.json', name: '马克思主义基本原理' },
  { file: 'math.json', name: '高等数学' },
];
```

### JSON 格式

通用字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| `sequence` | 否 | 题号 |
| `chapter` | 否 | 章节名，缺失时隐藏章节筛选 |
| `type` | 否 | 题型，缺失时根据答案自动推断 |
| `difficulty` | 否 | `easy` / `medium` / `hard`，缺失时隐藏难度筛选 |
| `question` | **是** | 题干文本 |
| `answer` | **是** | 答案（格式因题型而异） |
| `options` | 选择题**是** | 选项数组 |

#### 选择题（`single_choice` / `multiple_choice` / `true_false`）

```json
{
  "sequence": 1,
  "chapter": "第一章",
  "type": "single_choice",
  "difficulty": "easy",
  "question": "题干文本",
  "answer": "A",
  "options": [
    { "label": "A", "text": "选项A" },
    { "label": "B", "text": "选项B" },
    { "label": "C", "text": "选项C" },
    { "label": "D", "text": "选项D" }
  ]
}
```

- `type`：`single_choice` / `multiple_choice` / `true_false`，缺失时根据答案自动推断
- 多选题 `answer` 为多个字母（如 `"ACD"`）
- 判断题无需 `options`，答案为 `正确` 或 `错误`

#### 计算题（`calculation`）

支持多层计算，每问可设**精确值**或**范围**。支持内嵌数据表格。

```json
{
  "sequence": 1,
  "chapter": "第一章",
  "type": "calculation",
  "difficulty": "medium",
  "question": "根据以下数据计算：",
  "table": {
    "headers": ["商品", "基期价格", "基期数量", "报告期价格", "报告期数量"],
    "rows": [
      ["甲", "10", "100", "12", "120"],
      ["乙", "8", "150", "9", "180"]
    ]
  },
  "sub_questions": [
    { "id": 1, "text": "第一问：求 X 的值" },
    { "id": 2, "text": "第二问：求 Y 的值" }
  ],
  "answer": {
    "1": { "scope": [0.98, 1] },
    "2": { "value": 3.14 }
  }
}
```

- `table`（可选）— 数据表格，`headers` 为列名数组，`rows` 为数据行数组
- `sub_questions` — 各子问的题干，`id` 对应 `answer` 的键
- `answer` 字典中，`scope` 表示闭区间范围，`value` 表示精确值（误差 < 0.001）
- 全部答对计一次正确，部分错误计一次错误

#### 主观题（`subjective`）

基于结巴分词（segmentit）的词集对比评分。将参考答案和用户答案分别分词、去停用词，计算词重叠率。

```json
{
  "sequence": 1,
  "chapter": "第一章",
  "type": "subjective",
  "difficulty": "hard",
  "question": "结合实际，说明实践是检验真理的唯一标准。",
  "answer": {
    "reference": "实践是检验真理的唯一标准，因为：第一，真理的本性在于主观与客观相符合；第二，实践具有直接现实性的特点，能够把主观认识与客观实际联系起来加以对照；第三，只有实践才能证明认识是否具有真理性。脱离实践的思想理论和逻辑证明都不能作为检验真理的标准。",
    "min_match": 0.25
  }
}
```

- `reference` — 参考答案，用作评分基准（分词后与用户答案对比）
- `min_match` — 词重叠率阈值（命中词数 / 参考总词数），低于此值判错

> **向后兼容**：也支持旧版 `groups`（分组匹配）和 `keywords`（平铺关键词）格式，当没有 `reference` 时自动降级。

## 答题模式

| 模式 | 说明 |
|------|------|
| 重做错题 | 从错题本抽取，答对移除暂时错题，可选是否含长期记忆 |
| 闯关模式 | 答错100道即失败 |
| 无限模式 | 答对3次的题不再出现，错题反复出现；顶部显示"已答题 X/∞"；按钮"新一轮"可重置 |
| 限时模式 | 选章节/难度，自定义答题时间（分钟），10分钟内50题，查看答案时暂停计时 |

### 选项随机排列

每次出现题目时，选项 ABCD 的顺序会被随机打乱（原始 JSON 不变），只在渲染层重新编号；判定时以新编号为准。

### 计算题 / 主观题答题流程

- 渲染输入区（计算题：每问一个输入框；主观题：textarea）
- 用户填写后点击"提交"按钮
- 提交后显示逐问反馈/匹配详情
- 提交按钮隐藏，"下一题"按钮出现，用户手动点击进入下一题（不会自动跳转）

## 错题本

- **暂时错题**：答错自动加入，重做答对后自动移除
- **长期记忆**：用户主动加入 / 写备注时自动加入，重做答对不移除，需手动移除
- 点击错题本可浏览、分类切换（暂时/长期）、移除
- 底部按钮在窄屏（<480px）时自动重排：`<` `>` 在第一行水平分布，其余按钮垂直排列
- 答错的题目在错题本详情页可直接写备注（灰色输入框），保存后该题自动从"暂时错题"升为"长期记忆"

### 分析标签页

错题本顶部 chip 切换到蓝色"分析"标签后可见：

- **各章节错题数**（柱状图 · 降序）
- **迟疑最久的题目** Top 10（单位：秒），点击任一行可跳到该题详情
- 页面会实时反映当前答题进度（无需等待退出）
- 右下角有"清空分析数据"按钮

## 交互

- 单选/判断：点击即判定，1.5秒自动下一题
- 多选：点击切换选中，确认按钮提交
- 计算题/主观题：填写后点提交，手动点下一题
- 连击特效：连续3题答对闪绿光，10题掉落+Perfect，可在答题页 `+` 号开关
- 答题点 `+` 号可切换分类、写备注

## 音效

同目录放置以下 `.ogg` 文件即可启用音效（缺失时静默跳过）：

| 文件 | 触发时机 |
|------|---------|
| `orb.ogg` | 每次答对 |
| `break.ogg` | 每次答错（同时答题卡四周闪红光） |
| `levelup.ogg` | 累计答对次数达到 10 的倍数时（如第 10、20、30 题） |

## 备注

- 答题页 `+` 号面板、错题本详情页都可输入备注
- **写了备注的题目自动进入长期记忆**（并退出暂时错题）
- 备注在导出/导入进度 JSON 时会一起带走

## 进度管理

无限模式右上角齿轮 ⚙ 可：

- 导出/导入进度 JSON（包含无限模式进度、答题统计、自定义难度、错题本、备注、分析数据）
- 重置全部进度（含无限模式与分析数据）

不点击"新一轮"按钮时，进入无限模式会继承历史已答题数。
